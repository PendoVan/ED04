# Fase 02 — Diseño de Artefactos

En esta fase se detallan los requisitos (usuario, funcionales y no funcionales), se proponen los **mockups** de las interfaces clave y se define **pseudocódigo** para las reglas de negocio principales del sistema de reservas (bloqueos, tope semanal, ventana de cancelación). Esta fase da trazabilidad y prepara la base para el modelado de datos y casos de prueba.

## Archivos de la fase
- [`requisitos-usuario.md`](./requisitos-usuario.md) — Requisitos de Usuario (RU) alineados con la RTM.
- [`requisitos-funcionales-mandatorios.md`](requisitos-funcionales-mandatorios.md) — Requisitos Funcionales obligatorios (RF Must).
- [`requisitos-funcionales-mejora.md`](requisitos-funcionales-mejora.md) — Requisitos Funcionales de mejora (RF Should/Could).
- [`requisitos-no-funcionales.md`](requisitos-no-funcionales.md) — Requisitos No Funcionales (RNF).
- [`interfaces-usuario.md`](interfaces-usuario.md) — Mockups (3 pantallas) y criterios visuales.
- [`pseudocodigo.md`](pseudocodigo.md) — Lógica/validaciones clave (disponibilidad, reservar, cancelar).

> Las imágenes de mockups serán guardadas en `project/fase02/mockups/` (formato `.png`).
