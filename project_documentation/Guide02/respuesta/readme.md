# Respuesta a la Pregunta de Investigación  
## Sistema de Reservas de Canchas FISI — 2025-II

El modelo de ciclo de vida **en espiral** sí resulta aplicable para completar el desarrollo del prototipo del *Sistema de Reservas de Canchas FISI* en un periodo de tres meses durante el semestre académico 2025-II. Este modelo combina elementos iterativos con actividades de gestión de riesgos, lo cual se adapta adecuadamente al tiempo limitado y al carácter progresivo del proyecto.

En nuestro caso, el prototipo fue construido mediante fases sucesivas: análisis, diseño, modelado de procesos, arquitectura, implementación inicial del backend y frontend, y pruebas. Cada una de estas etapas se desarrolló de manera iterativa, donde se revisaron decisiones técnicas (como el uso de FastAPI, Prisma ORM y MySQL), se evaluaron riesgos (fallas en dependencias, errores de conexión, integración frontend-backend) y se replantearon componentes antes de avanzar. Esto coincide con las espirales del modelo.

Además, el alcance del proyecto —un sistema web con autenticación institucional, gestión de disponibilidad, reservas, notificaciones y administración básica— es adecuado para dividirse en ciclos pequeños de trabajo, donde cada iteración entrega un incremento del prototipo. De esta manera, en cada vuelta de la espiral se reduce la incertidumbre y se consolida un producto funcional.

Finalmente, el tiempo de tres meses es suficiente siempre que el proyecto se gestione por iteraciones cortas, priorizando requisitos críticos y atendiendo los riesgos a tiempo. El modelo en espiral permite justamente este enfoque progresivo y controlado, facilitando que un equipo de estudiantes complete el prototipo sin abandonar la calidad ni la coherencia técnica del sistema.

**Conclusión:**  
Sí, el modelo de ciclo de vida en espiral es aplicable y adecuado para completar el prototipo dentro del plazo establecido, ya que facilita la planificación iterativa, el control de riesgos y la evolución constante del sistema desarrollado.
